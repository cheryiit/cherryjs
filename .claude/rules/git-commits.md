# Git & Commit Rules

## Before Every Commit

1. Run `npm run test:arch` — must pass (788+ tests, 0 failures)
2. Run `npm run lint:deps` — must show 0 dependency violations
3. Run `npm run typecheck` — TypeScript must compile clean
4. Quick command: `npm run check` runs all three

**Never commit if any of these fail.** Fix the underlying issue, do not bypass with `--no-verify`.

## Commit Message Format

Conventional Commits style. Single subject line in present-tense imperative, then optional body.

```
<type>(<scope>): <subject>

<optional body explaining WHY, not WHAT>
```

### Types
- **feat** — new feature (new domain, new component, new lib module)
- **fix** — bug fix
- **refactor** — code restructure with no behavior change
- **test** — adding/changing tests
- **docs** — documentation only (CLAUDE.md, .claude/rules/, README)
- **chore** — config, tooling, dependency bumps
- **style** — formatting, no logic change
- **perf** — performance improvement

### Scope (optional but encouraged)
Use the affected layer or domain:
- `convex/users`, `convex/payments`, `convex/lib`, `convex/core`
- `app/routes`, `app/components`, `app/lib`
- `tests`, `arch`, `mcp`, `deps`

### Examples
```
feat(convex/payments): add Polar webhook deduplication
fix(convex/users): handle nullable email in createOrUpdate
refactor(app/lib): extract toast logic into withToast wrapper
test(arch): add admin-security enforcement suite
chore(deps): bump @convex-dev/better-auth to 0.11
docs(CLAUDE.md): add channel handler max-line rule
```

## Branch Naming

```
<type>/<short-description>
```

- `feat/polar-webhook-dedup`
- `fix/email-validator-edge-case`
- `refactor/extract-storage-helpers`
- `chore/upgrade-tanstack-router`

Avoid: `my-changes`, `wip`, `temp`, `bugfix-1234`. Be descriptive.

## What NOT to Commit

- `.env`, `.env.local`, `.env.dev`, `.env.prod` — secrets, never
- `.claude/settings.local.json` — personal harness overrides
- `CLAUDE.local.md` — personal instructions
- `node_modules/`, `dist/`, `.output/`, `.vinxi/`
- `convex/_generated/` — auto-regenerated, but Convex docs say commit it. **Commit it.**
- `app/routeTree.gen.ts` — auto-generated by TanStack. **Commit it.**
- Any file with API keys, tokens, or webhook secrets in it

## PR Workflow

- Branch from `main`, never commit to main directly in shared repos
- One logical change per PR — refactor + feature in separate PRs
- PR title follows commit message format
- PR description: WHY this change, not WHAT (the diff shows what)
- All architectural tests must pass on CI before merge
- If you change a public Convex function signature, document the migration path

## Risky Operations — Always Confirm First

These need explicit user approval, never run autonomously:

- `git push --force` (any variant) — overwrites remote history
- `git reset --hard` — discards uncommitted work
- `git rebase -i` — interactive, not supported in agent context
- `git checkout .` / `git restore .` — discards local changes
- `git branch -D` — deletes branches that may have unmerged work
- `git clean -f` — deletes untracked files
- `git commit --amend` after pushing — rewrites published history
- Pushing to `main` / `master` directly
- Deleting any tag

## Co-Author Attribution

When AI generates commits on behalf of the user, include co-author trailer:

```
Co-Authored-By: Claude <noreply@anthropic.com>
```

This is set automatically by Claude Code via `attribution` in `.claude/settings.json` if configured.
